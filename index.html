Product Overview

NB UI Kit is a comprehensive design system built on neo-brutalism principles. It provides over 20 fully accessible components with consistent theming via CSS custom properties. Every element has been designed for maximum clarity and visual impact.

Key Features

Zero dependencies. Pure CSS and vanilla JavaScript. Tree-shakable components that you can import individually or all at once. Full ARIA compliance with keyboard navigation built into every interactive component.

Responsive from mobile to ultra-wide. Dark mode ready. Custom theming via tokens. RTL support planned for v2.

Simple Pricing

NB UI Kit is free and open source under the MIT license. Use it in personal projects, commercial applications, and client work without any restrictions.

NB UI Kit is a clean, accessible design system built on neo-brutalism principles. It provides production-ready components with bold borders, hard-offset shadows, and vibrant accent colors. Everything is built with pure CSS and vanilla JavaScript with zero external dependencies.
Simply include the CSS and JS files in your project. Add <link rel="stylesheet" href="css/nb-all.css"> in your head and <script src="js/nb-all.js"></script> before the closing body tag. Components initialize automatically via data attributes.
Yes. Every interactive component includes proper ARIA attributes, keyboard navigation, focus management, and screen reader support. Modals trap focus, dropdowns support arrow key navigation, and all form elements use semantic HTML with proper labeling.

Single-Open Accordion

Absolutely. All colors are defined as CSS custom properties in the tokens layer. Override any --nb-* variable to theme the entire system at once. You can even define multiple themes by toggling classes on the root element.
Dark mode support is built into the token architecture. Override the color tokens under a .dark class or @media (prefers-color-scheme: dark) query to create a dark theme.
NB UI Kit supports all modern browsers: Chrome, Firefox, Safari, and Edge. The CSS uses progressive enhancement with sensible fallbacks. CSS layers ensure predictable specificity across the entire system.
