Visually Hidden

Hides elements visually while keeping them accessible to screen readers. The focusable variant reveals the element on focus.

Usage

The .nb-sr-only class visually hides content but keeps it in the accessibility tree for screen readers.

Add .nb-sr-only--focusable to make it appear when focused (useful for skip links).

Example markup:
<label class="nb-sr-only" for="search">Search</label>
<input id="search" type="text" placeholder="Search..." />

<!-- Focusable variant (skip link) -->
<a class="nb-sr-only nb-sr-only--focusable" href="#main">
  Skip to main content
</a>

Focus Trap

Traps keyboard focus within a container. Tab cycles through focusable elements without escaping. Essential for modals and dialogs.

Usage

Focus Trap is a JS-only utility (no CSS). It wraps any container to keep Tab focus cycling within it.

Example markup:
<div data-nb-focus-trap>
  <input type="text" placeholder="Name" />
  <input type="email" placeholder="Email" />
  <button>Submit</button>
</div>

<!-- Start inactive -->
<div data-nb-focus-trap data-nb-focus-trap-active="false">
  ...
</div>

<!-- Programmatic control -->
<script>
  el._nbFocusTrap.activate();
  el._nbFocusTrap.deactivate();
</script>

Portal

Teleports child content to document.body or a custom target, escaping overflow and stacking-context constraints. Used for modals, tooltips, and overlays.

Usage

Portal is a JS-only utility (no CSS). It moves children out of the DOM tree to the end of document.body or a specified container.

Example markup:
<!-- Default: teleports to body -->
<div data-nb-portal>
  <div class="nb-modal-backdrop">...</div>
</div>

<!-- Custom target -->
<div data-nb-portal data-nb-portal-target="#overlay-root">
  <div class="nb-tooltip">...</div>
</div>

<!-- Restore content programmatically -->
<script>
  el._nbPortalRestore();
</script>
