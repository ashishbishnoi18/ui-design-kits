Docs / Introduction

Introduction

DevKit is a dark-first, zero-dependency UI component library built for developer tools and applications.

Overview

DevKit provides 156+ ready-to-use components, from inputs and tables to full page layouts. It uses semantic CSS custom properties for theming and native @layer directives for clean specificity management.

Tip
DevKit requires no build tools or JavaScript frameworks. Include the CSS and JS files, then use the component classes in your HTML.

Features

  • 156 components across 9 categories — Layout, Typography, Forms, Data Display, Navigation, Feedback, Marketing, Application, Utility
  • Dark-first design — gorgeous dark theme by default with a polished light theme
  • Zero dependencies — pure CSS + vanilla JS, no frameworks needed
  • CSS @layer based architecture — clean specificity, easy overrides
  • Accessible — ARIA attributes, keyboard navigation, focus management
  • 12 sample pages — landing, dashboard, login, settings, and more

Design Philosophy

DevKit prioritizes developer experience. The dark theme uses #050505 base with #10b981 emerald accent — easy on the eyes for long coding sessions. Components use minimal borders (1px solid), subtle surfaces, and clean typography with Inter and JetBrains Mono fonts.

Docs/ Installation

Installation

Add DevKit to your project in under a minute.

CDN (easiest)

Add these two lines to your HTML:

html
<link rel="stylesheet" href="https://cdn.jsdelivr.net/gh/ashishbishnoi18/devkit@main/dist/devkit.min.css">
<script src="https://cdn.jsdelivr.net/gh/ashishbishnoi18/devkit@main/dist/devkit.min.js"></script>

Download

Copy the dist files into your project:

html
<link rel="stylesheet" href="devkit.min.css">   <!-- 241 KB -->
<script src="devkit.min.js"></script>           <!-- 194 KB -->

npm

bash
npm install devkit-ui

Then include in your HTML:

html
<link rel="stylesheet" href="node_modules/devkit-ui/dist/devkit.min.css">
<script src="node_modules/devkit-ui/dist/devkit.min.js"></script>

Development (source)

For development with individual component files:

html
<link rel="stylesheet" href="devkit/css/dk-all.css">
<script src="devkit/js/dk-core.js"></script>
<script src="devkit/js/dk-all.js"></script>

Build from Source

bash
git clone https://github.com/ashishbishnoi18/devkit.git
cd devkit
npm run build
# → dist/devkit.css + dist/devkit.min.css
# → dist/devkit.js  + dist/devkit.min.js
Docs/ Usage

Usage

How to use DevKit components in your HTML.

Basic Pattern

All DevKit components use the dk- class prefix. Sub-elements use a single underscore: dk-card_header. Variants use flat naming: dk-card--hover.

html
<div class="dk-card dk-card--hover">
  <div class="dk-card_header">
    <div class="dk-card_title">Project Alpha</div>
  </div>
  <div class="dk-card_body">
    <p>Status: Active</p>
  </div>
</div>

Interactive Components

Components with JavaScript behavior use data-dk-* attributes for auto-initialization:

html
<!-- Accordion — auto-inits via data-dk-accordion -->
<div class="dk-accordion" data-dk-accordion="single">
  <div class="dk-accordion_item">
    <button class="dk-accordion_trigger">Section 1</button>
    <div class="dk-accordion_content">
      <div class="dk-accordion_body">Content here</div>
    </div>
  </div>
</div>

<!-- Modal — trigger from anywhere -->
<button data-dk-modal-open="my-modal">Open Modal</button>
<div class="dk-modal-backdrop" id="my-modal" data-dk-modal>
  <div class="dk-modal">
    <div class="dk-modal_header">
      <div class="dk-modal_title">Confirm</div>
      <button class="dk-modal_close">&times;</button>
    </div>
    <div class="dk-modal_body">Are you sure?</div>
  </div>
</div>

State Classes

DevKit uses is-* classes for component states:

  • is-open — dropdowns, modals, drawers
  • is-active — tabs, nav links, stepper steps
  • is-checked — checkbox cards, radio cards
  • is-disabled — inputs, buttons (also works with [disabled])
  • is-loading — buttons, containers with loading overlay
  • is-visible — tooltips, toasts

Events

Interactive components emit custom events with the dk: prefix:

js
// Listen for modal events
document.querySelector('#my-modal').addEventListener('dk:modal-open', () => {
  console.log('Modal opened!');
});

// Listen for select changes
document.querySelector('[data-dk-select]').addEventListener('dk:select-change', (e) => {
  console.log('Selected:', e.detail.value);
});

// Programmatic toast
DK.toast({ message: 'Saved!', type: 'success', duration: 3000 });

Responsive Classes

Grid and visibility utilities support responsive prefixes:

  • sm: — 640px+
  • md: — 768px+
  • lg: — 1024px+
html
<div class="dk-grid dk-grid-cols-1 sm:dk-grid-cols-2 lg:dk-grid-cols-4">
  <div class="dk-col-span-1">...</div>
</div>
Docs/ Theming

Theming

Customize DevKit's look with CSS custom properties and theme switching.

Dark & Light Themes

DevKit defaults to dark. Switch to light by setting data-theme="light" on <html>:

html
<html data-theme="dark">  <!-- default -->
<html data-theme="light"> <!-- light mode -->

Theme Toggle

Use the built-in theme toggle component:

html
<button data-dk-theme-toggle>Toggle Theme</button>

This saves the user's preference to localStorage under the key dk-theme.

Design Tokens

Override any semantic token in your own CSS to customize the entire library:

css
:root {
  /* Change accent from emerald to blue */
  --accent:       #3b82f6;
  --accent-hover: #2563eb;
  --accent-muted: rgba(59, 130, 246, 0.12);
  --accent-text:  #60a5fa;

  /* Change fonts */
  --font-sans: 'Geist', system-ui, sans-serif;
  --font-mono: 'Geist Mono', monospace;

  /* Softer radius */
  --dk-radius-sm: 6px;
  --dk-radius-md: 10px;
  --dk-radius-lg: 14px;
}

Docs/ DK.register()

DK.register()

Register a component initializer function.

Signature

js
DK.register(name: string, initFn: (el: HTMLElement) => void): void

How It Works

The name maps to elements with a data-dk-<name> attribute. When DK.init() runs (automatically on DOMContentLoaded), it finds all matching elements and calls initFn once per element.

A WeakSet prevents double-initialization — the same element is never initialized twice, even if DK.init() is called again.

Example

js
;(function (DK) {
  DK.register('counter', function (el) {
    var count = 0;
    var display = DK.$('.counter-value', el);
    var btn = DK.$('.counter-btn', el);

    DK.on(btn, 'click', function () {
      count++;
      display.textContent = count;
      DK.emit(el, 'dk:counter-change', { count: count });
    });
  });
})(window.DK);
html
<div data-dk-counter>
  <span class="counter-value">0</span>
  <button class="counter-btn">+1</button>
</div>
Docs/ DK.init()

DK.init()

Scan DOM and initialize all registered components.

Signature

js
DK.init(root?: Document | HTMLElement): void

Usage

Called automatically on DOMContentLoaded. Also called automatically via MutationObserver when new nodes are added to the DOM. You only need to call it manually if you bypass normal DOM insertion.

js
// Re-init a specific container after innerHTML change
var container = document.getElementById('dynamic-area');
container.innerHTML = newHTML;
DK.init(container);

// Init a single component type
DK.initComponent('accordion', container);
Docs/ DK.trapFocus()

DK.trapFocus()

Trap keyboard focus within a container for modals and overlays.

Signature

js
DK.trapFocus(container: HTMLElement): () => void  // returns release function

Usage

js
// Trap focus
var release = DK.trapFocus(modalElement);

// Tab key will cycle through focusable elements inside modalElement
// Focus moves to first focusable element automatically

// Later — release the trap and restore previous focus
release();

Used internally by Modal, Drawer, Command Palette, and Alert Dialog components. The trap intercepts Tab and Shift+Tab keydown events, wrapping focus between the first and last focusable elements.
